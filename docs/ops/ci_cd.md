# CI/CD

## Цели
- Быстрая обратная связь
- Повторяемые релизы

## Рекомендуемые стадии
1. Lint & Format
2. Unit Tests
3. Build
4. Integration/E2E (по месту)
5. Security (SCA)
6. Release (теги/релизы)

## Примечания
- Кэшировать зависимости
- Артефакты сборки — сохранять
- Секреты — через CI секреты
